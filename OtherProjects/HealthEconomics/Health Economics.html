Automating Credit Card & Online Payment Data Transformation

Description:

Each day, credit card and online payment files are uploaded into HRC (internal reconciliation system). These files need to be transformed and validated to match daily bank deposits before being loaded. Previously, this process involved manual file cleanup, multiple copy-paste steps, and a lot of attention to avoid mismatches between payment reports and bank data. Thanks to automation, the file processing time was reduced from 1 hour to just 5 minutes, significantly increasing efficiency and allowing staff to focus on higher-value tasks.

Solution

I developed an automated workflow using Power Query and VBA macros to streamline this daily process.

Key Features

  • Dynamic Parameters in Power Query :The tool uses dynamic file paths and parameters to automatically switch between new data sources each day ‚Äî no need to manually update queries.
  • Data Transformation & Cleanup Raw payment files are cleaned and formatted through Power Query to meet HRC‚Äôs required structure.
  • Validation Control An additional control step was added to verify that transformed totals match daily bank deposit records, helping catch discrepancies early.
  • VBA Macros for Triggering A lightweight macro helps trigger the query refresh and final export to make the process one-click for end users.

Result

  • Time Saved: Reduced daily processing time significantly (exact hours depend on volume)
  • Accuracy Improved: Automated control step ensures payment data aligns with bank deposits before upload
  • Ease of Use: Designed for non-technical users ‚Äî all they need to do is drop in the new file and click a button
  • Scalability: Works with any number of daily files thanks to dynamic parameters

Tech Stack

  • Power Query (with parameterized data sources)
  • Excel VBA (for automation)
  • HRC (internal reconciliation system)
  • Daily bank reports for validation
