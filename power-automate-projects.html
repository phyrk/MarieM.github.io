Power Automate Projects

Automation workflows built with Power Automate to streamline business processes across Microsoft 365 and other platforms.

Description:

This Power Automate flow automates the end-to-end processing and approval of customer refund requests submitted by the operations team through an Excel-based form. The flow supports multiple payment types (Credit Card, Online Payments, Checks) and routes requests dynamically to different approval groups based on the amount and method of payment. It ensures data consistency, prevents duplicate requests, and streamlines multi-level approvals via Microsoft Teams and SharePoint integration.

Business Scenario

In a real-world customer service environment, refund requests were previously managed manually via email and Excel files, leading to delays, missed approvals, and duplicate processing. With this solution:

  • ✅ Eliminates manual processing and email chains
  • ✅ Prevents duplicate refunds via validation logic
  • ✅ Reduces approval turnaround time by routing to the correct group instantlyc
  • ✅ Creates a clear audit trail in SharePoint

    • Refund Request Flow

    Step 1: Data Retrieval

  • Manual Trigger via HTTP Request — activated directly from a button in an Excel form.
  • Retrieves refund request data and approver configuration from multiple Excel tables.
  • Supports tiered logic based on:

    • Refund amount
    • Payment method
    • CC recipients

    Step 2: Duplicate Detection

    Checks existing entries in SharePoint to prevent duplicate refund processing using key fields:

    • Customer ID
    • Amount
    • Date

    If a duplicate is found:

    • Sends an email notification to the requester
    • Stops the flow — no further processing

    Step 3: Request Processing

    📝 If the request is unique, it gets logged in SharePoint.

    📦 Routes the request to the correct approval group based on the payment method:

    • 💳 Credit Card
    • 💻 Online Payments
    • 🧾 Check

    Each group is dynamically defined via the Excel config.

    Step 4: Approval Workflow

    📨 Sends approval requests using:

    • Microsoft Teams Adaptive Cards
    • Or email-based approvals (if Teams is not used)

    📌 Captures decisions (Approve / Reject) and logs results in SharePoint.

    Step 5: Notifications & Logging

    ✅ On Approval:

    • Updates SharePoint item status to: Approved
    • Sends a confirmation email to the requester

    ❌ On Rejection:

    • Updates SharePoint item status to: Rejected
    • Sends an email with rejection reason
